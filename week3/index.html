Object methods: "this"

  • Objects
  • A function is called a method when it is a property of an object. Such as:
    let user = {name: "John", age: 30};
    user.sayHi = function() {
    alert("Hello!");
    };
    user.sayHi(); // Hello!
    The method is sayHi and the object is user
  • The shorter syntax for methods within an object literal is usually thoe most preferred which looks like this:
    user = { sayHi: function() { alert("Hello"); } };
    user = { sayHi() { // same as "sayHi: function(){...}" alert("Hello"); } };
    Which means you can leave out the keyword "function" and just put sayHi()
  • You can access the object through the method with the keyword "this".
  • The value of "this" is the object before the dot "." which is the one used to call the method. For example within the sayHi function you put: alert(this.name);
  • "This" can be used in any function even it is not a method of an object
  • Arrow functions though do not have their "own" this

Javascript: Novice to Ninja
Chapter 5: Objects

  • In JavaScript an object is a self-contained set of related values and function. The act as a collection that is of named propertes mapped to any JavaScript value like strings, numbers, booleans, arrays and functions.
    It is called a method if the property's value is a function.
  • An object is like a dictionary where you can look up a property name and then see a value.
  • Objects are usually used when wanting to keep related information and functionality in the same place together.
  • An object literal is an object that is created by wrapping all its properties and methods in curly braces {}
    All objects can be changable at any time when the program is running which means that its properties and methods can be removed or changed and then if wanted new properties can be added to the object even if it was declard using a "const".
  • You can create an object literal like this: const spiderman = {};
  • You access the properties of an object by using the dot notation. Like so: superman.name
    Outputs: << 'Superman'
    You can also access it by using bracket notation. Like so: superman['name']
    Outputs: << 'Superman'
  • The dot notation is the most common way but the bracket notation does have a few advantages: like it is the only way to access nonstandard property and method names that aren't following the variable nameing rules. As well as it lets you evaluate an expression and use it as the property key. For example:
    superman["real" + " " + "name"] // the property is built using string concatenation
    << "Clark Kent"
    If you try to access a property that doesn't exist though, undefined will be returned. For example:
    superman.city
    << undefined
  • When Calling Methods we also can use dot or bracket notation. Calling a method is the same when we invoke a function so parentheses need to be placed after the method name. For example:
    superman.fly()
    << 'Up, up and away!'
    superman['fly']()
    << 'Up, up and away!'
