Chapter 8: CSS3 Transforms and Transitions

  • Transform properties in CSS3 let you translate, rotate, scale and/or skew any element on the page.
    You can manipulate an element's appearance using transform functions. The transform property value is one or more transform functions that are separated by spaces which will be applied in the order they'r provided.
  • Translation functions allows you to move elements left, right, up or down. These functions behave similarly to the "position: relative;" when declaring top and left, moving elements up and down or left and right along the x and y axes.
    When a translation function is employed, you're are moving elements without impacting the flow of the document.
    While "position: relative" allows you to position an element either against its current position or against a parent or other ancestor, translated elements can only be moved relative to its current positon.
  • The translate(x,y) function moves an element x from the left, and y from the top:
    transform: translate(45px, -45px);
  • Example for the ad, they want the word "dukes" to move over to the right when the user hovers over it...the HTML is

    <h1>Put your <span>dukes</span> up, sire</h1>

    CSS is:
    .ad-ad2 h1:hover span {
    color: #484848;
    transform: translateX(40px);
    }
    But this doesn't work because "transforms" don't work on inline elements.But we can fix that by just adding "display: inline-block;"
    .ad-ad2 h1:hover span {
    color: #484848;
    transform: translateX(40px);
    display: inline-block;
    }
  • The scale(x,y) function can scale an element by the defined factors, first horizontally then vertically. If there is only one value provided, it will be used fo both the "x" and "y" values, growing or shrinking your element or pseudo-element while maintaining the original aspect ratio.
    For example, scale(1) would leave the element the same size, scale(2) would double its proportions, and scale(0.5) would halve the size, and so on.
    Providing different values will distort the element, as you'd expect:
    transform: scale(1.5, 0.25);
  • But scale has no impact on the document flow which means if you were to scale inline-block elements, text that is around it will fail to accommodate it with reflowing (not allowing space between, kind of overlapping the text)
    Because of this if need be you could try adjusting the elements height, width, or font-size instead of using a scale() transform.
  • The rotate()function rotates an element around the point of origin by a specified angle value given. Same as scale, the default point of origin is element's center.
    Usually, angles are declared in degrees, positive degrees moving clockwise and negative moving counterclockwise.
    In addition to degrees, valeus can also be provided in grads, radians, or turns.
    Rotate example below:

    .ad-ad2 h1:hover span {
    color: #484848;
    transform: rotate(10deg) translateX(40px) scale(1.5);
    }

    This rotates the span by ten degrees clockwise.
    The rotate is declared before the transalte so that is applied first because transforms are applied in the order provided so what you want done first you put first.
    The span will be rotated 10 degrees, then moved 40px along the rotated x axis.
  • To animate an element in CSS, you hav to first create a named animation, then attach it to an element in that element's property declaration block.
    Animations themseleves don't do anything which in order to animate an element, you have to associate the animation with that element.
  • In order to create an animation, you use the @keyframes rule. Include @-webkit-keyframes for all WebKit implementations followed by a name that you choose, which serves as the identifier for the animation. Then, you can specify your keyframes.
    For example, an animation called "myAnimation", the @keyframes rule would look like this:

    @-webkit-keyframes myAnimation {
    /* put animation keyframes here */
    }
    @keyframes myAnimation {
    /* put animation keyframes here */
    }
    Don't quote the animation name.
