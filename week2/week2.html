Notes

  • Automatic Semicolon Insertion (ASI) is used by JavaScript interpreters which places semicolons at the end of each line for you.
    Although that can cause errors, so it's best to put a semi-colon and each statement on a new line.
  • Blocks which is a series of statements within curly brackets do not need to be terminated by a semicolon.

Variables

  • Variables must be declared before they can be used.
  • JavaScript keywords to declare variables are: const and let
  • const is used when the variable will not be changed to another value.
  • let is used if the variable might change later in the program.
  • The "=" operator is used to assign a value to a variable or constant (const)
  • When using "const" you can't reassign the value if it is primitive data type like a string
    but if it is a non-primitive data type like an object then it can change
    This is known as mutating the object. For example:
    const name = { value: 'Sam' };
    name.value = 'Heather';
  • const can not be reassigned to another object though. For example:
    const name = { value: 'Sam' };
    Tried to reassign another object
    name = { value: 'Heather' };
    It can't be changed.
  • Using const helps with making your program more predictable because the value can't change.

Scope

  • const and let are block scoped which means their value only exist within the block they were declared in.
  • "var" worked alot like how "let" works but "var" does not have block scope.
  • global scope is any variable that is declared outside of a block which it means it can be accessible in the whole entire program.
    This is not best practice because you can run into the same name with another global variable and could overwrite each other's values.
  • local scope are created within blocks.
    Which means any variable declared inside that block is only accessible within that block

Template Literals

  • Template Literals are special types of strings
  • They use the backtick character ` `. For example: `Hello world!`;
  • An expression can be within the template literal and the value will be displayed. This is called interpolation. For example:
    const name = `Sam`;
    `Hello ${name}!`
    Output: 'Hello Sam!'

Converting

  • The best way to convert strings to numbers is by using the Number(' '); function.
    If the string can't be converted into a number then "NaN" is returned as a result.
  • You can also multiply a numerical string by 1 which will convert into a number. This is called type coercion.
  • Another way is to put the plus sign "+" operator in front of the numerical string. For example:
    const answer = +'5';
  • The best way to convert numbers to strings is by using the String(' '); function.
  • You can also add an empty string which is type coercion. For example:
    3 +''; Returns: '3'
  • Another option is the toString() method. For example:
    10..toString(); Returns: '10'
