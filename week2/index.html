Variables

  • Variables must be declared before they can be used.
  • JavaScript keywords to declare variables are: const and let
  • const is used when the variable will not be changed to another value.
  • let is used if the variable might change later in the program.
  • The "=" operator is used to assign a value to a variable or constant (const)
  • When using "const" you can't reassign the value if it is primitive data type like a string
    but if it is a non-primitive data type like an object then it can change
    This is known as mutating the object. For example:
    const name = { value: 'Sam' };
    name.value = 'Heather';
  • const can not be reassigned to another object though. For example:
    const name = { value: 'Sam' };
    Tried to reassign another object
    name = { value: 'Heather' };
    It can't be changed.
  • Using const helps with making your program more predictable because the value can't change.

Scope

  • const and let are block scoped which means their value only exist within the block they were declared in.
  • "var" worked alot like how "let" works but "var" does not have block scope.
  • global scope is any variable that is declared outside of a block which it means it can be accessible in the whole entire program.
    This is not best practice because you can run into the same name with another global variable and could overwrite each other's values.
  • local scope are created within blocks.
    Which means any variable declared inside that block is only accessible within that block

Template Literals

  • Template Literals are special types of strings
  • They use the backtick character ` `. For example: `Hello world!`;
  • An expression can be within the template literal and the value will be displayed. This is called interpolation. For example:
    const name = `Sam`;
    `Hello ${name}!`
    Output: 'Hello Sam!'

Converting

  • The best way to convert strings to numbers is by using the Number(' '); function.
    If the string can't be converted into a number then "NaN" is returned as a result.
  • You can also multiply a numerical string by 1 which will convert into a number. This is called type coercion.
  • Another way is to put the plus sign "+" operator in front of the numerical string. For example:
    const answer = +'5';
  • The best way to convert numbers to strings is by using the String(' '); function.
  • You can also add an empty string which is type coercion. For example:
    3 +''; Returns: '3'
  • Another option is the toString() method. For example:
    10..toString(); Returns: '10'

Parsing Numbers

  • parseInt() can also be used to convert a string that is a (representaion) of a numerical value back into a number. For example:
    parseInt('1010,2'); the result is: 10. Or parseInt('omg, 36'); the result is: 31912. parseInt('23',10); the result is: 23.
  • If the parseInt function starts with a number, it will ignore any letters that are after the numbers.
  • Also, if you use parseInt with decimal, it will remove anything that is after the decimal point.
  • parseFloat converts a string that has a number and a decimal point and converts into a floating point decimal number. For exmaple:
    parseFloat('2.9',10); the result is: 2.9.

Array Methods

  • An array is a typeof 'object' and is an ordered list of values. You can create an array either an array literal like "const myArray = [];" or an array constructor function like "const myArray = new Array();".
    but the best way to create an area is an array literal
  • To remove the last item of an array, you can use the pop() method. For example: nameOfArray.pop(); It returns the last item of that array but it is no longer includes that item.
  • To remove the first item of an array, you can use the shift() method. It works in a similar way as the pop() method.
  • The push() method appends a new value you choose to the end of the array. For example:
    nameOfArray.push('Flower'); It then returns a value which is the length of the array that includes that new value you just pushed.
  • The unshift() appends a new item to the start of the array. It works similarly to the push() method.
  • The concat() method is used to merge an array with another array which can be one or more arrays. For example:
    nameOfArray.concat(['Sun','Garden', 'Rain']); then returns:['Grass', 'Water', 'Dirt', 'Sun', 'Garden', 'Rain']
    The first 3 water, dirt and sun are from the original array but now the last three are merged in that original array.
  • Instead of the concat() method you could use the new spread operator which is three dots "..." that are placed in front of the array that can spread out the elements of the array and puts them together which then creates a new array. For example:
    nameOfArray = [ ...nameOfArray, ...['Sun','Garden', 'Rain'] ];
    Returns: ['Grass', 'Water', 'Dirt', 'Sun', 'Garden', 'Rain']
  • The join() method turns the array into a string and then each item in the array is separated by commas. For example:
    nameOfArray.join(); then returns: 'Grass, Water, Dirt, Sun, Garden, Rain'
    You can choose a different separator other than a comma like an ampersand sign as follows:
    nameOfArray.join(' & '); returns the same as above just instead of commas it is the ampersand sign.

    nameOfArray.reverse(); then returns the array but in reverse.
  • You can also sort the order of an array by using the sort() method. For example:
    nameOfArray.sort(); which sorts the array alphabetically.
    Numbers also can be sorted alphabetically but by their first digit not numerically where 9 would come after 10.

Functions

  • To define a function literal you would use a function declaration. It starts with the function keyword then the function name, followed by paraentheses and curly brackets that will contain your code. For example:
    function functionName() { console.log('Hello World!');}
  • Another way you can define a function literal is by creating a function expression which assigns an anonymous function to variable. For example:
    const goodbye = function() {console.log('Goodbye World!');};
    You can also create a named function expression as well which would just have the name of the function and then paraentheses after the function keyword.
  • The semicolon at the end of the (function expression) finishes the statement but a normal function declaration ends in a block which you don't need semicolons at the end of blocks.
  • Function() Constructors which the body of the function would be entered as a string. For Example:
    const hello = new Function('console.log("Hello World!");');
    Although it is not recommended to declare functions this way.
  • Arrow functions (arrow syntax) is a new way to declare functions which makes declaring functions more concise by using less wordy syntax.
  • Arrow functions are identified by the 'arrow' symbol "=>". Parameters come before the arrow and the main body of the function comes after the arrows.
  • Arrow functions are always anonymous so in order to refer back to them you have to assign them a variable.
  • Example of an arrow function: const add = (x,y) => x + y;
    If the function doesn't require any parameters a pair of empty parentheses have to still go before the arrow.
  • Functions are first-class objects in JavaScript which means they behave the same way as every other object. So functions can also be given be used as a parameter to another function. This is known as a callback.
  • You can implement a callback by adding a callback as a parameter and then invoking in inside the body of the function.
